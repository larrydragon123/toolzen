## 2026-05-20：建议功能完善 — 从入口到后台的完整闭环

### 为什么要改动

上一次提交实现了基本的建议表单和后台查看功能，但存在几个明显问题：

1. **用户找不到建议入口**：只有页面底部 Footer 里有一个小链接，首页和导航栏都没有任何提示。用户如果不滚动到底部根本不知道可以提交建议。
2. **搜索无结果时没有引导**：用户在搜索框输入一个不存在的工具名后，只看到"未找到匹配的工具"，然后就没了。这个时机恰恰是用户最可能想提建议的时刻，我们却没有抓住。
3. **后台管理页面体验差**：Admin 页面是纯手写 CSS 的独立页面，没有使用 Tailwind 的配色体系，看起来像另一个网站。而且中文内容在渲染时可能出现编码问题。
4. **API 安全性不足**：密码通过 URL 查询参数（`?password=xxx`）传递，会出现在服务器日志中。没有速率限制，可以被恶意刷数据。
5. **首页 CTA 按钮文字颜色异常**：全局 `<a>` 标签的颜色样式（unlayered CSS）优先级高于 Tailwind 的 `text-white`，导致按钮上白色文字被蓝色覆盖，几乎看不到。

### 怎么改的

#### 1. 搜索栏添加建议入口（`SearchBar.astro` + `SearchModal.tsx`）

在主页中间的大搜索框下方增加了一个隐藏的建议区块，当用户输入关键词且没有任何工具卡片匹配时，自动显示"未找到匹配的工具"和"建议此工具 →"按钮。顶部的 Ctrl+K 搜索弹窗也做了同样处理——在空白状态和搜索结果为空时都显示建议入口。

#### 2. 首页 CTA 区块（`index.astro`）

在"为什么选 ToolZen"板块和博客板块之间插入了一个醒目的渐变背景卡片，标题"没找到你需要的工具？"，引导用户点击"建议新工具 →"。Header 导航栏右侧也加了一个灯泡图标 💡 的建议按钮。

#### 3. 修复链接颜色问题（`BaseLayout.astro`）

根本原因是一行 CSS：
```css
a { color: var(--color-zen-500); }
```
这行样式不在任何 CSS Layer 中，属于 unlayered 样式，优先级高于 Tailwind utilities 所在的 layer。所以 `text-white` 等工具类无法覆盖它。

修复方式：
```css
@layer base {
  a { color: var(--color-zen-500); }
}
```
将样式放入 `@layer base`，让 Tailwind utilities 的优先级高于它。

#### 4. 重写 Admin 后台页面（`admin.astro`）

完全重写，导入 Tailwind CSS 直接使用工具类，配色和间距与主站完全一致。顶部加了导航栏（带 ToolZen logo 和 Admin 标识）。登录页面居中卡片设计。管理面板有统计卡片（全部/待处理/已查看/已实现）、状态选择器、复制和删除功能。空状态有友好的引导提示。

#### 5. API 安全增强

- **认证方式**：优先从 `Authorization: Bearer <token>` header 读取密码，兼容旧的 `?password=` 查询参数（向后兼容，但不推荐）
- **速率限制**：同一 IP 每小时最多提交 3 条建议，防止滥用
- **输入校验**：工具名最长 100 字符，描述最长 2000 字符，拒绝空值
- **新增接口**：`DELETE /api/suggestions?id=xxx` 删除建议，`PATCH /api/suggestions` 更新状态

#### 6. 部署配置

- 通过 `wrangler kv namespace create` 在 Cloudflare 创建了 KV 命名空间
- 配置了随机生成的管理员密码
- 添加了 `@cloudflare/workers-types` 类型定义以消除 TypeScript 报错

### 最终效果

- 用户可以从**三个入口**发现建议功能：首页 CTA 卡片、Header 导航、Footer 链接
- 搜索时找不到工具，直接引导用户建议
- 后台管理页面与主站风格统一，支持状态流转（待处理→已查看→已实现）和删除
- API 具备基本的安全防护（Bearer 认证、速率限制、输入校验）
- 部署到 Cloudflare Pages 后可立即使用

## 2026-05-20：工具建议表单和后台管理（初版）

### 为什么要做

ToolZen 目前有 20 多个工具，但我们并不清楚用户真正需要什么。很多工具网站都缺少一个让用户表达需求的渠道。这次改动让用户可以提出想要的功能，管理员可以在后台统一查看和管理这些建议。

### 做了什么

1. **建议表单页面**（`/zh/suggest/`、`/en/suggest/`）：包含工具名称、功能描述、邮箱（选填）三个字段，表单提交通过 JavaScript fetch 调用 API
2. **Cloudflare Pages Function API**：
   - `POST /api/suggest`：接收建议，存入 Cloudflare KV
   - `GET /api/suggestions`：管理员查看建议列表，需密码认证
3. **后台管理页面**（`/admin`）：密码登录后展示所有建议，包含时间、邮箱、IP 信息
4. **国际化**：中英文翻译完整覆盖建议表单和后台

### 技术选型

选择 Cloudflare KV 而非传统数据库，因为：
- 建议数据量不会很大（几百条量级）
- 不需要建表和 ORM，开发效率高
- Pages Functions 原生支持，零额外配置

## 2026-05-19：英文博客和 SEO 优化

### 背景

中文博客上线后，发现国际用户完全无法阅读。为了提升 AI 搜索引擎（如 Perplexity、Google SGE）的发现率，需要英文内容。

### 改动

- 将 5 篇中文博客翻译为英文，使用 Astro 内容集合管理双语内容
- 优化 `llms-full.txt` 文件，包含完整工具描述和博客摘要，提升 AI 爬虫对网站的理解
- 首页增加博客展示区块
- 修复博客列表链接缺少 `/blog/` 路径的 bug
